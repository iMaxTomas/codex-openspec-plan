# 思路进化说明

## 起点问题

高质量的 shell 白板解释很容易停留在“某次对话刚好画得好”，一旦没有固化成 skill，后续就会退回普通段落或僵硬模板。

## 第一阶段误区

- 试图固定成一张唯一版式
- 把输出写死成 `L0/L1/L2`
- 只关心画得像不像，不关心 operator-first flow

## 收敛后的抽象

最终把 skill 固化成：
- semantic building blocks
- layout selection system
- operator-first explanation flow
- correction anchors
- anti-patterns

## 为什么值得复用

这份计划适合所有想把“表达方式”产品化的人。它不是单张图的模板，而是一套“何时用哪种 shell 语义布局”的规则。

## 迁移建议

- 保留 layout selection，而不是照抄具体版式
- 保留 `not this, but this` 的纠偏锚点
- 把你自己领域的高频误解补成反模式
