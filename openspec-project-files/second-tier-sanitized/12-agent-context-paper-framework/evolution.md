# 思路进化说明

## 起点问题

很多团队会直接开始写 `AGENTS.md`、rules 或 skills，但没有先建立“哪些结论来自讨论、哪些来自论文、哪些已经足够进入长期规则”的研究框架。

## 第一阶段误区

- 直接根据直觉改长期配置
- 论文阅读和本地讨论彼此脱节
- 把研究过程留在聊天里，不回写成可复用计划

## 收敛后的抽象

先固定研究顺序：
- 本地讨论材料
- 论文源码或等价可深读输入
- 对照框架
- 后续 OpenSpec 方向

## 为什么值得复用

这份包不只是研究两篇论文，而是把“怎么从本地讨论走到可复用 agent context 框架”固定成了一个 OpenSpec 研究 workflow。

## 迁移建议

- 把 `<paper-reading-skill>` 换成你自己的 source-faithful 阅读工具
- 准备你自己的两份本地讨论材料
- 保留“先讨论、再论文、再回写 OpenSpec”的顺序
